External Prerequisites

Move the following images into your private registry

  1. quay.io/domino/domino-kc-groups-based-irsa-mapper:prod-v1
  2. quay.io/domino/cloud-identity-management-utils:prod-v1

Create an IAM Role for the admin service

The service which manages your IRSA mappings needs an IAM role with permissions to update trust policies. Add the following policy:

Note: Update the account ID and role pattern as needed. 
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "iam:UpdateAssumeRolePolicy",
                "iam:GetRole"
            ],
            "Resource": "arn:aws:iam::11111111111:role/acme_domino_*"
        }
    ]
}

Add the following trust policy to allow the service account to assume this role:

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Federated": "arn:aws:iam::11111111111:oidc-provider/oidc.eks.us-west-2.amazonaws.com/id/AAAAAAAAAA"
            },
            "Action": "sts:AssumeRoleWithWebIdentity",
            "Condition": {
                "StringLike": {
                    "oidc.eks.us-west-2.amazonaws.com/id/AAAAAAAAAA:sub": "system:*:domino-field:domino-irsa-lite-admin-sa",
                    "oidc.eks.us-west-2.amazonaws.com/id/AAAAAAAAAA:aud": "sts.amazonaws.com"
                }
            }
        }
    ]
}

Installation Prerequisites

Create Namespace

export ns=domino-field
kubectl create namespace ${ns}

# Label Namespace for Domino Compute
kubectl label namespace ${ns} domino-compute=true

Create a Keycloak Client

Get your Keycloak admin password (username is usually keycloak):

export ps=domino-platform
kubectl get secret -n $ps keycloak-http -o jsonpath='{.data.password}' | base64 --decode && echo

Create a Keycloak client with minimum privileges:

export KC_URL="http://keycloak-http.domino-platform.svc.cluster.local:80/auth"
export REALM="DominoRealm"
export KC_ADMIN_USER="keycloak"
export CLIENT_ID="irsa-mapping-manager"
#export KC_ADMIN_PASS="<ADD_ADMIN_PASS>"
python ./scripts/bootstrap_keycloak_client.py | jq

This will generate output like:

{
  "kc_url": "https://<DOMINO_URL>.cs.domino.tech/auth",
  "realm": "DominoRealm",
  "client_id": "irsa-mapping-manager",
  "client_secret": "j5...",
  "granted_roles": ["query-groups", "query-users", "manage-users"]
}
Note: Save the client_secret - you'll need it for the Helm values.

Helm Installation

Install

Update client_secret in your values file, then:

export ns=domino-field
helm install cloud-identity-management-utils helm/cloud-identity-management-utils/ \
    -n $ns -f helm/cloud-identity-management-utils/myvalues.yaml

Upgrade

export ns=domino-field
helm upgrade cloud-identity-management-utils helm/cloud-identity-management-utils/ \
    -n $ns -f helm/cloud-identity-management-utils/myvalues.yaml

Delete

export ns=domino-field
helm delete cloud-identity-management-utils -n $ns

Configure User Groups

export KC_URL="http://keycloak-http.domino-platform.svc.cluster.local:80/auth"
export REALM="DominoRealm"
export KC_ADMIN_USER="keycloak"
export CLIENT_ID="irsa-mapping-manager"
export KC_GROUPS_CONFIG="./resources/kc_groups.json"

# Dry run to see the changes
python ./scripts/kc_apply_group_spec.py --spec ${KC_GROUPS_CONFIG} --dry-run

# Apply the changes if you like them
python ./scripts/kc_apply_group_spec.py --spec ${KC_GROUPS_CONFIG}

IRSA Configuration

1. Create a default role for all users

Create a trust policy that allows all service accounts in domino-compute namespace:

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Federated": "arn:aws:iam::11111111111:oidc-provider/oidc.eks.us-west-2.amazonaws.com/id/<ID>"
            },
            "Action": "sts:AssumeRoleWithWebIdentity",
            "Condition": {
                "StringLike": {
                    "oidc.eks.us-west-2.amazonaws.com/id/<ID>:aud": "sts.amazonaws.com",
                    "oidc.eks.us-west-2.amazonaws.com/id/<ID>:sub": "system:serviceaccount:domino-compute:*"
                }
            }
        }
    ]
}
Warning: Do not add any policies to this role. This is only to test that IRSA is configured correctly by AWS webhooks.

2. Configure managed IAM roles

Update trust policies for roles managed by Keycloak groups. The sub claim will be populated by the sync process:

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Principal": {
                "Federated": "arn:aws:iam::11111111111:oidc-provider/oidc.eks.us-west-2.amazonaws.com/id/<ID>"
            },
            "Action": "sts:AssumeRoleWithWebIdentity",
            "Condition": {
                "StringLike": {
                    "oidc.eks.us-west-2.amazonaws.com/id/<ID>:aud": "sts.amazonaws.com",
                    "oidc.eks.us-west-2.amazonaws.com/id/<ID>:sub": ""
                }
            }
        }
    ]
}

Sync Commands

Sync Domino Users to K8s Service Accounts

export DEFAULT_AWS_ROLE_ARN="arn:aws:iam::946429944765:role/acme_domino_default_role"

# Dry run first
python ./scripts/sync.py users --default-aws-role-arn $DEFAULT_AWS_ROLE_ARN --dry-run

# Apply
python ./scripts/sync.py users --default-aws-role-arn $DEFAULT_AWS_ROLE_ARN

Sync Keycloak Groups to IAM IRSA

export DOMINO_API_PROXY="http://localhost:8899"

# Dry run first
python ./scripts/sync.py irsa --dry-run

# Apply
python ./scripts/sync.py irsa

Summary

This is a five-step process:

  1. Create Keycloak client with minimum privileges
  2. Install the helm chart for cloud-identity-management-utils
  3. Configure Keycloak groups and map them to IAM roles with IRSA trust policies
  4. Sync Domino users to K8s Service Accounts with proper annotations
  5. Sync Keycloak groups to IAM roles with IRSA trust policies

Steps (1) and (2) need to be done only once.

Steps (3), (4) and (5) need to be repeated as needed:

  • New user added to Domino: Update JSON spec in step (3) to include the user in a Keycloak group. Run step (4) to create a K8s Service Account, then run step (5) to update IAM role mappings.
  • New IAM role needed: Update the JSON spec in step (3) and run step (5) again.
  • User changes groups: Update the JSON spec in step (3) and run step (5) again.
